# Skill: benchmark-analysis

Decide whether a benchmark belongs and where to place it.

## Steps

1. **Identify** — Name, canonical source (repository or paper), and what it measures.
2. **Relevance** — Does it evaluate AI systems for bioengineering tasks (structure, sequence, generation, property prediction, docking, fitness, etc.)?
3. **Quality** — Is it maintained, documented, and used by the community? Does it have clear tasks and metrics?
4. **Placement** — Usually Benchmarks and Evaluation; place under a modelling section only if it is primarily a model with an attached benchmark.
5. **Overlap** — Check for duplication with existing benchmark entries.
6. **Safety** — Confirm the description avoids operational biological detail.

## Output

- Decision: include / exclude, with reason.
- If include, the entry and target section, noting what it measures in one neutral line.
