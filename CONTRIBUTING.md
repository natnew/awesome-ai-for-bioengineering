# Contributing

Thank you for helping keep this list high quality. This is a curated Awesome List, so contributions are reviewed for relevance, source quality, and safety rather than accepted by default.

## What belongs

- Resources directly relevant to **AI engineering for bioengineering**: biological foundation models, protein language models, structure and function prediction, generative protein design, molecular generation, DNA/RNA modelling, biomolecular simulation, AI agents for science, self-driving laboratories, datasets, benchmarks, tooling, and responsible release practices.
- Canonical sources: GitHub repositories, papers (arXiv or DOI), official documentation, institutional or project pages.
- Technically meaningful resources with durable value.

## What does not belong

- Marketing pages, thin landing pages, or low-signal tools.
- Duplicate entries or dead links.
- Resources unrelated to AI for bioengineering.
- Any wet-lab protocol, pathogen engineering guidance, genetic-manipulation procedure, synthesis route, or other operational biological experimentation detail. See [SECURITY.md](SECURITY.md).

## Entry format

Add entries to `README.md` only, in the appropriate section, keeping alphabetical order where practical:

```
* [Name](https://example.org) - Neutral, technically precise description.
```

Optional plain-text tags may be added where they improve scanability, for example `` `paper` `` `` `model` `` `` `code` ``. Do not over-tag.

## Source and description quality

- Use HTTPS canonical links. Prefer the official repository, paper, or project page over secondary coverage.
- Keep descriptions to one neutral line. Avoid hype, superlatives, and claims the source cannot substantiate.
- Be precise about what the resource is (paper, model, benchmark, dataset, tool, platform, lab, company).

## How to suggest a resource

1. Open a **Resource suggestion** issue, or
2. Open a pull request adding a single entry.

Please:

- Submit **one resource per pull request** where possible.
- Explain briefly why the resource belongs and why it is high signal.
- Confirm the link is canonical and the entry contains no unsafe procedural detail.

The maintainer may edit descriptions for clarity, neutrality, and consistency, or decline entries that weaken the list.
